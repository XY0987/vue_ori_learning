## 7. 初始化/启动流程分析模板

**适用于：** 任何有多阶段启动过程的项目（CLI 工具、Web 应用框架、服务端框架、桌面应用等）。

很多项目都有复杂的初始化流程，涉及多阶段、并行优化、安全门控等。用此模板做系统化分析。

```
### 🚀 初始化流程分析：[项目名/模块名]

#### 设计模式总结
一句话概括初始化的核心设计模式（如：多阶段 + memoize 幂等 + 并行预取优化）

#### 初始化流程图
[Mermaid 图，用 subgraph 分隔每个阶段]

#### 阶段 N：[阶段名] (`文件名`, 行数)

> **关键设计**：此阶段的核心设计理念。

| 序号 | 操作 | 做了什么 | 设计意图 | 产出 |
|------|------|----------|----------|------|
| N.1 | `函数名()` | 详细说明 | 为什么 | 产出 |
| N.2 | ... | ... | ... | ... |

#### 阶段 N 优秀实现方式

| 技术/模式 | 实现方式 | 效果 | 可借鉴场景 |
|-----------|----------|------|------------|
| ... | ... | ... | ... |

[重复以上结构 for 每个阶段]

#### 初始化流程设计亮点总结
- 亮点1：...
- 亮点2：...
```

**识别初始化阶段的常见模式：**

| 阶段类型 | 常见操作 | 典型设计 |
|----------|----------|----------|
| 环境预设 | 环境变量、heap size、feature flags | 放在模块最顶部，利用 ESM 同步评估 |
| 快速路径分发 | --version/--help 等无需完整初始化的命令 | 动态 import 避免加载完整模块 |
| 配置加载 | 读取配置文件、合并多源配置 | memoize 幂等 + 分层配置 |
| 网络/安全初始化 | TLS 证书、代理、mTLS、API 预连接 | 严格顺序（证书→代理→预连接） |
| 信任/权限门控 | 工作区信任对话、权限模式选择 | 信任前后分阶段释放能力 |
| 会话/状态初始化 | 会话 ID、工作目录、后台服务 | fire-and-forget 异步 + 关键路径同步 |
| 后台预热 | 缓存预热、插件加载、索引构建 | 延迟到首次渲染后执行 |

## 8. 优秀实现方式提取模板

**适用于：** 任何源码分析的阶段。这是一个独立的分析维度——不仅回答"怎么做的"，更回答"好在哪里"。

```
### ✨ 优秀实现方式汇总：[模块名/步骤名]

| 技术/模式 | 实现方式 | 效果 | 可借鉴场景 |
|-----------|----------|------|------------|
| 模式名称 | 具体怎么做的（函数名、代码位置、关键逻辑） | 量化或定性效果 | 什么场景可以复用 |
```

**常见优秀实现分类速查：**

| 类别 | 常见模式 | 识别信号 |
|------|----------|----------|
| **启动性能** | 零导入快速路径、ESM 副作用并行 I/O、动态 import 延迟加载、API 预连接、早期输入捕获 | `await import()`、模块头部副作用、`preconnect`、`prefetch` 关键词 |
| **安全设计** | 分层权限、信任前/后分阶段能力、fail-closed 默认值、配置快照防篡改 | `checkPermissions`、`trust`、`deny rules`、`safe` vs `full` 变量名 |
| **弹性恢复** | 多级递进恢复、错误暂扣(withheld)、熔断退化、补偿重试 | `recovery`、`fallback`、`withheld`、`retry`、`escalate` 关键词 |
| **并发优化** | 无锁状态机、读写自动分区、投机执行、流式管道 | `guard`、`concurrent`/`serial`、`speculative`、`streaming` 关键词 |
| **状态管理** | memoize 幂等、using 自动 dispose、原子更新、fire-and-forget | `memoize`、`using`、`void fn()`、`state = {...state}` 模式 |
| **缓存策略** | 多层缓存、预热与惰性、LRU 淘汰、cache scope 分级 | `cache_control`、`prefetch`、`LRU`、`ephemeral`/`global` 关键词 |
| **可扩展性** | 插件/hook 系统、条件编译 DCE、分层配置、注册表模式 | `hook`、`plugin`、`feature()`、`register`、`buildTool()` 工厂函数 |
